## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Pipeline → Logger | Pipeline passes file_id, trace, row_number, error messages to logger |
| Logger → Log output | Log output may contain partner data traces or error messages from untrusted files |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Information Disclosure | process_file() → emit_row_failed | mitigate | Reason field comes from ValidationError (already sanitized by Validator); trace field truncated to 256 chars in logger (per T-07-01) |
| T-07-06 | Information Disclosure | process_file() → emit_file_failed | mitigate | Error message from exception may contain stack traces; logger should truncate error to 512 chars to prevent log flooding |
| T-07-07 | Denial of Service | Per-row logging in tight loop | accept | Logging to stdout is buffered; per-row log is O(1) and does not block pipeline; rate limiting is log aggregator concern |
| T-07-08 | Repudiation | Missing log events on crash | mitigate | FILE_FAILED emitted in exception handler with best-effort; if logger itself fails, pipeline continues (logger errors caught and suppressed) |
</threat_model>
