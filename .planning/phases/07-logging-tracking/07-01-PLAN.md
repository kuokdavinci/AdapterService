    - emit_row_failed(file_id, row_number, trace, reason) → logs ROW_FAILED with error details
    - When log_format="text", output is human-readable; when "json", output is one JSON object per line
    - All log events include: timestamp, level, event_type, file_id (when applicable)
    - Per-row events additionally include: row_number, trace, status, reason (on failure)

Create src/logging/ package with StructuredLogger implementation:

1. **src/logging/__init__.py** — Export StructuredLogger and LogEventType.

2. **src/logging/logger.py** — Implement:

   a. **LogEventType(StrEnum)** with values: FILE_STARTED, FILE_COMPLETED, FILE_FAILED, ROW_SUCCESS, ROW_FAILED

   b. **JSONFormatter(logging.Formatter)**:
      - Override format() to produce JSON string
      - Include: timestamp (ISO 8601), level, event, message, and all extra fields from record
      - Use json.dumps with default=str for Decimal/datetime serialization
      - Filter out internal logging._record fields (name, pathname, lineno, etc.)

   c. **TextFormatter(logging.Formatter)**:
      - Override format() to produce readable string
      - Format: "[LEVEL] {event}: {message} {extra_fields}"
      - Extra fields as key=value pairs

   d. **StructuredLogger**:
      - __init__(name: str = "reconciliation"): gets singleton logger via logging.getLogger(name)
      - _setup_handler(): creates StreamHandler(sys.stdout), sets formatter based on settings.log_format ("json" → JSONFormatter, else → TextFormatter), sets level from settings.log_level
      - _log_event(event_type: LogEventType, extra: dict): creates LogRecord with event type in extra, logs at INFO level
      - emit_file_started(file_id: str, file_name: str, partner: str): calls _log_event with FILE_STARTED, includes file_id, file_name, partner
      - emit_file_completed(file_id: str, total: int, success: int, failed: int, duration_ms: float): calls _log_event with FILE_COMPLETED, includes stats and duration
      - emit_file_failed(file_id: str, error: str): calls _log_event with FILE_FAILED, includes error
      - emit_row_success(file_id: str, row_number: int, trace: str): calls _log_event with ROW_SUCCESS, includes row_number, trace, status="SUCCESS"
      - emit_row_failed(file_id: str, row_number: int, trace: str, reason: str): calls _log_event with ROW_FAILED, includes row_number, trace, status="FAILED", reason
      - get_logger() → returns the underlying logging.Logger for direct use

   e. Use a module-level singleton: _logger = StructuredLogger() with get_structured_logger() accessor

   Do NOT add any new dependencies to requirements.txt — use only Python stdlib (logging, json, sys, datetime) plus existing settings module.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Application → Log output | Log fields could contain untrusted data from partner files (file_name, trace values) |
| Log output → Log aggregator | JSON log lines consumed by external systems (ELK, CloudWatch, etc.) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | JSONFormatter | mitigate | Sanitize/truncate file_name and trace fields to max 256 chars before logging; never log raw file content or PII |
| T-07-02 | Tampering | Log output stream | accept | Logs are append-only stdout/stderr; integrity ensured by log aggregator, not application |
| T-07-03 | Denial of Service | StructuredLogger.emit_row_* | mitigate | No per-row file I/O — logs go to stdout buffer; rate limiting handled by log aggregator if needed |
| T-07-04 | Spoofing | LogEventType enum | accept | Enum values are internal constants, not user-controllable input |
</threat_model>
