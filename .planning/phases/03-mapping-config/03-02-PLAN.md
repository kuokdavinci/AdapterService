<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MongoDB → ConfigLoader | Repository returns data from DB; loader validates before use |
| ConfigLoader → Cache | Loader stores validated configs in cache; cache trusts stored data |
| ConfigLoader → Downstream (ExcelStreamReader, Normalizer) | Loader returns validated MappingConfig; downstream trusts config is valid |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering | ConfigLoader | mitigate | Validation runs on every load (even cache miss); invalid configs rejected with ConfigLoadError before reaching cache or downstream |
| T-03-07 | Spoofing | ConfigLoader cache key | mitigate | Cache key uses FileType enum value (not raw string); partner/workflow_type validated by repository query |
| T-03-08 | Denial of Service | ConfigLoader | mitigate | ConfigLoadError includes validation_errors for debugging; no infinite retry loops; cache miss → single DB query |
| T-03-09 | Information Disclosure | ConfigLoader error messages | mitigate | ConfigLoadError.message is generic ("Config validation failed"); detailed errors only in validation_errors list (not logged automatically) |
| T-03-10 | Elevation of Privilege | ConfigLoader | accept | ConfigLoader only reads from MongoDB (no write/delete); repository is read-only for config loading |
</threat_model>
