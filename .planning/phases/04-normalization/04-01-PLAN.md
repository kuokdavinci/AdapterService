**NormalizationResult dataclass:**
- `data: dict[str, Any]` — successfully normalized canonical field values keyed by path
- `errors: list[ValidationError]` — all conversion/validation errors encountered

**TransactionNormalizer class:**
- `__init__(self, field_mappings: list[FieldMapping])` — store mappings, validate non-empty
- `normalize(self, row: dict[str, Any], row_number: Optional[int] = None) -> NormalizationResult` — main entry point

**Source field resolution (per FieldMapping):**
- If `fm.column` is set: look up `row[fm.column]` (e.g., row["A"])
- If `fm.sourceField` is set: look up `row[fm.sourceField]` (e.g., row["transaction_id"])
- If neither is set and type is not CONSTANT: produce ValidationError "no column or sourceField configured"
- CONSTANT type: skip row lookup entirely, use `fm.constant` directly

**Type conversion methods (all static, pure functions):**
- `_convert_string(value: Any, fm: FieldMapping, row_number: Optional[int]) -> tuple[str | None, ValidationError | None]`
  - Convert value to str. If value is None/empty string → ValidationError.
- `_convert_decimal(value: Any, fm: FieldMapping, row_number: Optional[int]) -> tuple[Decimal | None, ValidationError | None]`
  - Convert via Decimal(str(value)). If value is float → ValidationError ("float not allowed for monetary values"). If conversion fails → ValidationError with original value in reason.
- `_convert_date(value: Any, fm: FieldMapping, row_number: Optional[int]) -> tuple[datetime | None, ValidationError | None]`
  - If value is already datetime → return as-is.
  - If value is string → try parsing with formats: "%Y-%m-%d", "%d/%m/%Y", "%Y-%m-%d %H:%M:%S", "%d/%m/%Y %H:%M:%S". First match wins.
  - If no format matches → ValidationError with original value and attempted formats in reason.
- `_convert_constant(fm: FieldMapping, row_number: Optional[int]) -> tuple[str | None, ValidationError | None]`
  - Return fm.constant directly. If fm.constant is None/empty → ValidationError.

**normalize() method flow:**
1. Initialize result = NormalizationResult(data={}, errors=[])
2. For each FieldMapping in self._field_mappings:
   a. Resolve source value from row (or skip for CONSTANT)
   b. Call appropriate _convert_* method based on fm.type
   c. If conversion succeeds → result.data[fm.path] = converted_value
   d. If conversion fails → result.errors.append(error)
3. Return result

**Error handling pattern:**
- Never raise exceptions — all errors collected as ValidationError
- Each ValidationError includes: field=fm.path, reason=descriptive message, row=row_number
- Continue processing all mappings even after errors (no fail-fast)

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Partner Excel → Normalizer | Untrusted cell values (strings, numbers, dates) flow through type converters |
| MappingConfig → Normalizer | Partner-specific field mappings could be malformed or malicious |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | _convert_decimal | mitigate | Reject float input explicitly (Decimal(str(value)) pattern); InvalidOperation caught and converted to ValidationError — no raw exceptions escape |
| T-04-02 | Tampering | _convert_date | mitigate | Whitelist date formats only (4 specific strptime patterns); no regex or eval-based parsing; unmatched formats produce ValidationError |
| T-04-03 | Tampering | normalize() source resolution | mitigate | KeyError on missing column/sourceField caught and converted to ValidationError; no raw exception propagation |
| T-04-04 | Denial of Service | normalize() hot path | mitigate | All converters are O(1) — no regex compilation, no network calls, no file I/O; 4 date format attempts max per DATE field |
| T-04-05 | Information Disclosure | ValidationError.reason | mitigate | Error reasons are descriptive but do not include raw cell values that might contain PII; only field name and error type exposed |
| T-04-06 | Tampering | MappingConfig field_mappings | accept | ConfigValidator (Phase 03) already validates structural integrity of field_mappings before Normalizer receives them; Normalizer trusts validated config |
</threat_model>
