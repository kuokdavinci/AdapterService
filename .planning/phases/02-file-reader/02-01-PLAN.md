<interfaces>
<!-- openpyxl read-only mode contracts (from documentation) -->

```python
from openpyxl import load_workbook

# Load in read-only mode for constant memory usage
wb = load_workbook(filename='file.xlsx', read_only=True)

# Access sheet by name
ws = wb['SheetName']

# List all sheet names
wb.sheetnames  # ['Sheet1', 'Sheet2', ...]

# Iterate rows as tuples of cell values
for row in ws.iter_rows(values_only=True):
    # row is a tuple: (cell_value_1, cell_value_2, ...)
    pass

# MUST close workbook after reading
wb.close()
```

Key behaviors:
- `read_only=True` enables lazy loading with near-constant memory consumption
- Cells returned are `ReadOnlyCell` objects, not standard `Cell`
- `values_only=True` yields plain Python values (str, int, float, datetime, None) instead of cell objects
- Workbook MUST be explicitly closed after processing
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| File system → ExcelStreamReader | Untrusted Excel files could contain malformed data, macros, or crafted content |
| ExcelStreamReader → Downstream pipeline | Row data must be clean tuples; None values for empty cells must be handled |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | ExcelStreamReader.__init__ | mitigate | Validate file extension (.xlsx/.xlsm only), verify file exists before opening, reject non-Excel files |
| T-02-02 | Tampering | File read operation | transfer | File integrity (hash verification) is reconciliation_file's responsibility, not the reader's |
| T-02-03 | Information Disclosure | ExcelStreamReader | mitigate | Reader does NOT log row data or cell contents — only logs file-level events (open, close, sheet count) |
| T-02-04 | Denial of Service | Workbook loading | mitigate | Use read_only=True for constant memory consumption regardless of file size; no full workbook loading |
| T-02-05 | Elevation of Privilege | Context manager | mitigate | Workbook is always closed in __exit__ even on exception; no file handle leaks possible |
</threat_model>
