```python
# iter_rows(values_only=True) yields tuples like:
# ('Header1', 'Header2', 'Header3')    — header row
# ('A', 'B', 'C')                       — data row
# (None, None, None)                    — empty row (all cells None)
# ('A', None, 'C')                      — partially filled row (NOT empty)
# ('Total:', 1000, '')                  — summary row
# ('', '', '')                          — empty row (all empty strings)
# ('Grand Total', 5000, None)           — footer row
```

Empty row detection: A row is empty if ALL cells are None or empty string.
Partially filled rows (some cells have values) are NOT empty — they are data rows.

  <action>Add row filtering to `ExcelStreamReader.iter_rows()`:

1. Update `__init__` to accept new optional parameters:
   - `skip_empty_rows: bool = True` — skip rows where all cells are None or empty string
   - `skip_patterns: list[str] | None = None` — list of string patterns; skip rows where ANY cell contains a pattern match (case-insensitive)

2. Add `_is_empty_row(self, row: tuple) -> bool` static method:
   - Returns True if ALL cells in the row are None or empty string ("")
   - Returns False if ANY cell has a non-empty value
   - Example: `(None, "", None)` → True (empty); `("A", None, "")` → False (has data)

3. Add `_should_skip_row(self, row: tuple) -> bool` method:
   - If `skip_empty_rows` is True and `_is_empty_row(row)` → return True
   - If `skip_patterns` is provided, check each non-None cell value:
     - Convert cell to string, check if any skip_pattern is a substring (case-insensitive)
     - Common patterns: "total", "grand total", "summary", "footer", "合计", "总计"
     - If any pattern matches → return True
   - Otherwise return False

4. Update `iter_rows()` generator to filter rows:
   - For each row from `self._worksheet.iter_rows(min_row=self._start_row, values_only=True)`:
     - If `_should_skip_row(row)` → continue (skip this row)
     - Otherwise → yield row

5. Common skip patterns for reconciliation files (use as default if skip_patterns is None):
   - `["total", "grand total", "summary", "footer", "合计", "总计", "小计"]`
   - These are case-insensitive substring matches against cell string values

6. Do NOT add MappingConfig integration — that comes in task 2.</action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Excel file content → Row filtering | Malicious files could craft rows to evade skip patterns or trigger pattern matching on legitimate data |
| MappingConfig → ExcelStreamReader | Misconfigured MappingConfig could cause reader to skip wrong sheet or wrong rows |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering | Row filtering logic | mitigate | Skip patterns are substring matches only (not regex), limiting injection risk; patterns are configurable per partner via MappingConfig, not hardcoded |
| T-02-07 | Denial of Service | Empty row detection | mitigate | `_is_empty_row` is O(n) per row where n = column count; no regex or expensive operations in hot path |
| T-02-08 | Repudiation | Row skipping | mitigate | Reader does NOT log which rows were skipped; if audit trail needed, downstream pipeline should log skipped row numbers |
| T-02-09 | Information Disclosure | from_mapping_config | mitigate | factory method only reads sheet_name and start_row from config — does NOT log or expose field_mappings or other sensitive config data |
</threat_model>
